# 넥스트스텝 프론트 연동 계약서

프론트가 백엔드와 주고받는 것 전부를 한 장에 압축. 상세는 `api-spec.md`·`frontend-spec.md`, 이 문서는 연동 계약의 핵심만.

## 도메인 (프론트가 알아야 할 개념)

```
Site(자리, PNU) ──1:N── Unit(물건) ──1:N── Tenancy(이력)
```

- **Site(자리)**: 지번 하나 = 한 건물/필지. 키 = `pnu`(19자리 문자열)
- **Unit(물건)**: 자리 안의 개별 점포. 대부분 자리=물건 1:1(실데이터상 호실분리 거의 없음). 키 = `unitId`
- **Tenancy(이력)**: 한 물건에 시간순으로 들어온 각 사업장. 개업~폐업 한 구간. 이게 "연대기"의 각 칸
- **조회 식별자 = PNU**. 지번 검색 → PNU → 물건 → 이력

## 엔드포인트 3개

| # | 메서드·경로 | 화면 | 상권API 실시간호출 |
|---|---|---|---|
| 1 | `GET /api/sites/search?query={지번}` | 랜딩 검색 | 없음 |
| 2 | `GET /api/sites/{pnu}` | 건물 상세(물건 목록) | 없음 |
| 3 | `GET /api/units/{unitId}` | 물건 상세(타임라인+이력별 marketInfo) | **여기만** |

- 1·2는 순수 DB 조회 → 빠르고 항상 성공
- 3만 상권 API를 실시간 호출 → `marketInfo` 부분은 실패 시 일부 null (전체는 200 유지)

## 응답 타입 (TypeScript)

```ts
// ── 공통 ──
interface Disclaimer { dataAsOf: string; note: string; }  // 모든 조회 응답에 상시
interface ApiError { error: string; message: string; }

// ── ① search ──
interface Candidate {
  pnu: string;
  jibunAddress: string;
  roadAddress: string;
  latitude: number | null;
  longitude: number | null;
  unitCount: number;      // 이 자리의 물건 수
  closedCount: number;    // 이 자리 누적 폐업 수
}
interface SearchResponse { candidates: Candidate[]; }

// ── ② site 상세 ──
type LocationSource = "license" | "sangga_api" | "overlap_inferred";
interface UnitSummary {
  unitId: string;
  label: string;                       // '단일 점포' | 호수 | '물건 A'
  currentBusinessName: string | null;  // 현재 영업 가게, 공실이면 null
  currentStatus: "영업" | "공실";
  totalTenancyCount: number;
  closedCount: number;
  averageSurvivalMonths: number | null;
  industryDetail: string | null;       // 상가API 세부업종(현재 영업 물건만)
  locationSource: LocationSource;
}
interface SiteDetail {
  site: { pnu: string; jibunAddress: string; roadAddress: string;
          latitude: number | null; longitude: number | null; };
  units: UnitSummary[];
  disclaimer: Disclaimer;
}

// ── ③ unit 상세 ──
type EnrichmentSource = "sangga_api" | "license_only";
interface Statistics {
  totalTenancyCount: number;
  closedCount: number;
  averageSurvivalMonths: number | null;
  longestSurvivalMonths: number | null;
  shortestSurvivalMonths: number | null;
}
interface MarketInfo {
  isPlaceholder: boolean;              // 현재 항상 true
  leaseAreaSqm: number | null;         // 목업(소스 없음)
  depositKrw: number | null;           // 목업
  monthlyRentKrw: number | null;       // 목업
  keyMoneyKrw: number | null;          // 목업
  dailyFloatingPopulation: number | null; // 목업
  sameCategoryNearbyCount: number | null; // 유일한 실값(상가API), 실패 시 null
  vacancyRatePercent: number | null;   // 목업
  asOf: string;
}
interface Tenancy {
  tenancyId: string;
  businessName: string;
  category: string;              // 인허가 대분류(예: 음식_일반음식점) — 항상 존재
  subCategory: string;           // 인허가 소분류(예: 일반음식점) — 항상 존재
  industryDetail: string | null; // 상가API 세부업종 — 영업중 이력만
  licensedAt: string;            // YYYY-MM-DD
  closedAt: string | null;       // null = 영업중
  status: "영업" | "폐업" | "휴업";
  survivalMonths: number | null;
  closedAtEstimated: boolean;    // 현재 데이터셋은 폐업일자 실값이라 항상 false
  enrichmentSource: EnrichmentSource;
  marketInfo: MarketInfo;
}
interface UnitDetail {
  unit: { unitId: string; label: string; jibunAddress: string; roadAddress: string; };
  statistics: Statistics;
  timeline: Tenancy[];           // licensedAt 오름차순(과거→현재)
  disclaimer: Disclaimer;
}
```

## 화면별 사용 데이터

**① 랜딩** — `search(query)` → 후보 리스트(지번·도로명·물건수·폐업수). 클릭 → `/map?q=&pnu=`(2026-07-10 8차: 경로파라미터 아닌 지도+쿼리스트링 방식, `frontend-spec.md` §3 참조)

**② 건물 상세** — `getSite(pnu)` → 지도(좌표 null이면 "위치 없음") + 물건 카드 목록. 카드 클릭 → `/units/:unitId`

**③ 물건 상세** — `getUnit(unitId)` →
- 통계 카드 4개(거쳐간 가게/폐업/평균생존/최장·최단)
- 운영 타임라인(가로 바, 구간폭=생존개월 비례)
- "가게 자세히 보기" 드롭다운(tenancyId 선택) → 좌: 인허가정보(businessName/category/subCategory/기간/상태) · 우: marketInfo(sameCategoryNearbyCount만 실값, 나머지 5필드 목업 + "예시" 뱃지 + "실 데이터 연동 전 예시값입니다" 캡션 상시)

## 업종 3필드 구분 (헷갈리기 쉬움)

| 필드 | 출처 | 존재성 | 화면 |
|---|---|---|---|
| `category` | 인허가 대분류 | 항상 | 타임라인 기본 표시 |
| `subCategory` | 인허가 소분류 | 항상 | 물건 상세 클릭 시 표시 |
| `industryDetail` | 상가API 세부 | 영업중만 | 있으면 subCategory 대신 우선 표시 |

## 에러

| code | HTTP | 상황 |
|---|---|---|
| INVALID_QUERY | 400 | query 누락 |
| SITE_NOT_FOUND | 404 | pnu 없음 |
| UNIT_NOT_FOUND | 404 | unitId 없음 |
| INTERNAL_ERROR | 500 | 서버 오류 |

- 검색 빈 결과는 에러 아님 → `{ candidates: [] }` 200
- `marketInfo.sameCategoryNearbyCount` 산출 실패는 에러 아님 → 그 필드만 null, 200 유지
